# Make 工程管理器

# 0x01 简介

Make 是一个工程管理器，能根据文件时间戳自动发现更新过的文件从而减少编译工作量。它通过 读入 Makefile 文件的内容来执行大量的编译工作，极大地提高了项目开发和维护的工作效率。

# 0x01 Makefile 基本结构

一个 Makefile 中通常包含如下内容：

  1. 需要由 make 工具创建的目标体，通常是目标文件、可执行文件或是一个标签。
  2. 要创建的目标体所依赖的文件。
  3. 创建每个目标体时所需要运行的命令。

格式为：

target: dependency
	command
1
2

在 Makefile 中的每一个 command 前必须有 ”Tab“ 符，否则在运行make命令时会出错。

例子：

hello:hello.c
	gcc -o main hello.c
clean: # clean 是目标功能，不是文件。不需要依赖文件，所以没有依赖文件
	rm main
rebuild:clean hello # 要想实现 rebuild 目标，只需要实现这两个小目标即可。两个小目标即是先决条件。
1
2
3
4
5
make hello
make clean
make rebuild

3

若需要换行，需要在换行的地方添加 \ 。

当没有确定的目标时，Make 以 Makefile 文件的第一个目标作为这次运行的目标。

要使 Make 不打印出命令， 就在命令前 加一个 @ 符号

# 0x02 Makefile 变量

变量是在 Makefile 中定义的名字，用来替代一个文本字符串，该文本字符串称为该变量的值。

变量的值可以替代目标体、依赖体、命令等。 Makefile 中定义变量有两种方式：一种是递归展开方式，一种是简单扩展方式。

递归展开方式定义的变量是在引用该变量时进行替换的，如果包含了对其他变量的引用，则在引用该变量时一次性将内嵌的标量全展开。缺点：不能在变量后追加内容，可能导致无限循环。

简单扩展型变量的值在定义处展开，只展开一次。

VAR=VAR
VAR:=var
Make 中变量使用格式为 $(VAR)
1
2
3

建议变量名用小写字母，预留大写字母作为控制隐含规则参数或用户重载命令选项参数的变量名。

例子：

OBJS = main.o kbd.o command.o display.o \
		insert.o
edit: $(OBJS)
	gcc -o $(OBJS) main
	
clean:
	rm $(OBJS) main

Makefile 的变量分为 用户自定义变量、预定义变量、自动变量以及环境变量。

预定义变量和自动变量无需定义就可以使用，部分有默认值。

常见预定义变量

命令格式 含义
AR 默认值 ar
AS 默认值 as
CC 默认值 cc
CPP 预编译器的名称，默认值 $(CC) -E
CXX 默认值g++
RM 默认值 rm -f
ARFLAGS 库文件维护程序的选项
CPPFLAGS 汇编程序的选项
CFLAGS c 编译器的选项
CPPFLAGS c 预编译器的选项
CXXFLAGS c++ 编译器的选项

自动变量

命令格式 含义
$* 不包含扩展名的目标文件名称
$+ 所有的依赖文件， 空格分开
$< 第一个依赖文件的名称
$? 所有时间戳比目标文件晚的依赖文件，以空格分开
$@ 目标文件的完整名称
$^ 所有不重复的依赖文件
$% 如果目标是归档成员，则该变量表示目标的归档成员名称。

例子：

OBJS = main.o kbd.o command.o display.o \
		insert.o
RM = rm -f
CC = gcc
CFLAGS = -Wall -g
edit: $(OBJS)
	$(CC) $^ -o $@
main.o:main.c defs.h
	$(CC) $(CFLAGS) -c -o $@ $< 
kbd.o:kbd.c defs.h command.h
	$(CC) $(CFLAGS) -c -o $@ $< 
...
clean:
	$(RM) edit $(OBJS)

# 3.1 隐式规则

例子：

OBJS = main.o kbd.o command.o display.o \
		insert.o
		
RM = rm -f
CC = gcc

	$(CC) $^ -o $@
main.o:main.c defs.h
kbd.o:kbd.c defs.h command.h

省略了

1

因为 Make 的隐式规则指出，所有的 .o 文件都可以由 .c 文件使用命令 $(CC) $(CPPFLAGS) $(CFLAGS) -c file.c -o file.o 生成

在隐式规则中只能查找到相同文件名的不同后缀名文件，如 kang.o 文件必须由 kang.c 文件生成。

常见隐式规则目录

对应语言后缀名 规则
C 编译 ： ".c" 变为 ".o" $(CC) $(CPPFLAGS) $(CFLAGS) -c
C ++ 编译： ”.cc“ 或".c" 变为 ".o" $(CXX) -c $(CPPFLAGS) $(CXXFLAGS)

# 0x04 Makefile 假目标

关键字：.PHONY

.PHONY:clean
clean :
	rm *.o
1
2
3

采用 .PHONY 关键字声明一个目标后，Make 不会将其当作一个文件来处理，而只是当作一个概念上的目标。
